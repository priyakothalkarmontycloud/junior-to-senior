Always ground examples in frontend work: React, state management (Redux/Zustand/Context), component libraries, CSS approaches (Tailwind/CSS Modules/styled-components), routing, testing (Cypress/Playwright/Jest), TypeScript adoption decisions, etc.

**Why:** Priya is a frontend developer. Backend/infra examples (SQS, Celery, PostgreSQL, DynamoDB, Redis) are unfamiliar territory and block comprehension rather than aiding it.

**How to apply:** Before writing any lesson or explanation that uses a concrete example, ask "would a frontend dev encounter this?" If not, swap it for a frontend equivalent.
