# Strategic Programming Resources

## Knowledge

- [Shape Up — Ryan Singer (Basecamp, free online)](https://basecamp.com/shapeup)
  The most practical book on shaping work before it enters development. Use for: scoping,
  appetite-setting, preventing scope creep. Read chapters 1–5 first.

- [A Philosophy of Software Design — John Ousterhout](https://web.stanford.edu/~ouster/cgi-bin/book.php)
  The best book on managing complexity in software systems. Use for: system design,
  decomposition, module boundaries, naming. Required reading.

- [Staff Engineer — Will Larson](https://staffeng.com/book)
  Practical guide to technical strategy and influence at senior levels. Use for: technical
  direction, navigating org dynamics, writing engineering strategy.

- [Software Engineering at Google — Winters, Manshreck & Wright (free PDF)](https://abseil.io/resources/swe-book)
  How Google thinks about engineering at scale. Use for: code review culture, testing
  philosophy, dependency management, large-codebase practices.

- [The Pragmatic Programmer — Hunt & Thomas](https://pragprog.com/titles/tpp20/the-pragmatic-programmer-20th-anniversary-edition/)
  The foundational professional developer guide. Use for: career perspective, tool habits,
  design philosophy, professional discipline.

- [Designing Data-Intensive Applications — Martin Kleppmann](https://dataintensive.net/)
  Deep treatment of distributed systems and data architecture tradeoffs. Use for: any
  scope involving data storage, consistency, reliability, or scale.

- [Joel on Software — Joel Spolsky (free online)](https://www.joelonsoftware.com/)
  Timeless essays on software development craft. Use for: spec writing, estimating,
  hiring, managing technical debt. "Painless Functional Specifications" series is directly
  relevant to scoping.

- [Architecture Decision Records — Michael Nygard (original article)](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions)
  The original ADR proposal. Short, foundational. Use for: understanding the format and
  purpose before writing your first ADR.

## Wisdom (Communities)

- [Hacker News (news.ycombinator.com)](https://news.ycombinator.com/)
  High signal-to-noise technical discussion. Filter by "ask HN" for career and design
  questions. Best for: getting a pulse on what senior engineers care about.

- [r/ExperiencedDevs](https://reddit.com/r/ExperiencedDevs)
  Strictly moderated subreddit for senior developer perspectives. Use for: architectural
  war stories, career strategy, code review disagreements.

- [Software Engineering Stack Exchange](https://softwareengineering.stackexchange.com/)
  Structured Q&A on design, patterns, and practices. Use for: specific architectural
  questions with written, reasoned answers.

## Gaps

- No strong resource yet on AI-era scoping specifically (how to scope work when AI generates
  the implementation). This is an emerging area; worth watching.
- Need a good resource specifically on QA strategy (testing philosophy, not testing tools).
